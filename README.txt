AutoScope 0.3.0 beta
Руководство пользователя

AutoScope — это приложение для сбора, хранения и анализа объявлений автомобильного рынка.
Программа помогает запускать парсинг объявлений, сохранять найденные данные в локальную базу,
строить отчеты и при необходимости выполнять сценарии в фоновом режиме.

Это beta-версия. В ней уже можно пользоваться основными возможностями приложения, но некоторые
функции могут быть не полностью отполированы. Если что-то работает странно, сохраните описание
действий, скриншот ошибки и файлы из папки Logs.


1. Что нужно для запуска

Для обычного запуска не требуется Visual Studio.

Желательно:
- Windows 10 или Windows 11;
- установленный Google Chrome, если вы планируете запускать парсеры, работающие через браузер;
- распакованная папка AutoScope в обычное место на диске.

Рекомендуемый путь для распаковки:
C:\Apps\AutoScope
D:\AutoScope

Нежелательные места:
C:\Program Files
C:\Windows
папки OneDrive
запуск прямо из архива

Перед запуском обязательно распакуйте архив полностью. Не запускайте приложение из окна архива.


2. Первый запуск

Откройте папку AutoScope и запустите:

AutoScope.WpfClient.exe

При первом запуске приложение может открываться немного дольше обычного. Это нормально.

Если Windows покажет предупреждение SmartScreen, это связано с тем, что beta-сборка не подписана
цифровым сертификатом. Для тестовой версии это ожидаемо. Запускайте приложение только если получили
архив из доверенного источника.


3. Главное окно

В главном окне отображаются основные разделы AutoScope:

- базы данных;
- сценарии;
- отчеты;
- текущие и последние процессы.

Через главное окно можно перейти к созданию или выбору базы, запуску сбора данных, запуску анализа,
управлению сценариями и просмотру результатов.


4. Базы данных

AutoScope хранит данные в локальных SQLite-базах. База данных — это файл, в который сохраняются
объявления и история их проверок.

Обычно работа выглядит так:

1. Создайте новую базу или выберите существующую.
2. Запустите сбор данных.
3. После завершения сбора запустите анализ.
4. Откройте сформированный отчет.

Если база не выбрана или была удалена вручную, некоторые действия могут быть недоступны.


5. Сбор данных

Сбор данных выполняется через парсеры. Парсер — это модуль, который получает объявления с выбранного
источника и передает их в AutoScope.

При запуске сбора обычно нужно выбрать:
- базу данных;
- парсер;
- стартовый адрес или параметры источника;
- количество объявлений;
- размер пакета обработки.

Во время работы парсера приложение может показывать ход выполнения. Если сбор занимает некоторое
время, не закрывайте приложение до завершения процесса.

Важно: парсеры зависят от сайтов-источников. Если сайт изменил разметку, ограничил доступ или временно
не отвечает, сбор может завершиться с ошибкой. Это не всегда означает ошибку самого AutoScope.


6. Анализ и отчеты

После того как данные сохранены в базе, можно запустить анализ.

Анализатор читает данные из базы и формирует результат. Обычно результатом является HTML-отчет,
который можно открыть в браузере.

Отчеты сохраняются в папке:

Reports

Если отчет не открылся автоматически, его можно найти и открыть вручную.


7. Сценарии

Сценарий — это сохраненный набор действий для повторного запуска. Например, можно настроить регулярный
сбор данных по выбранному источнику.

Для сценария можно указать:
- название;
- тип действия;
- выбранный модуль;
- базу данных;
- параметры запуска;
- интервал автоповтора.

Если интервал равен 0, сценарий не запускается автоматически и используется только для ручного запуска.

Для автоматического запуска сценариев приложение должно быть запущено. Его можно оставить работать
в фоновом режиме через системный трей.


8. Работа в фоне и трей

AutoScope может продолжать работу после закрытия основного окна, если включена работа через трей.
В этом случае крестик закрытия скрывает окно, но приложение остается запущенным.

В трее можно:
- открыть AutoScope;
- открыть управление процессами;
- открыть настройки;
- полностью выйти из приложения.

Если нужны автоматические сценарии, не завершайте приложение через пункт "Выход" в трее.


9. Автозапуск вместе с Windows

В настройках можно включить запуск AutoScope вместе с Windows.

Это удобно, если вы хотите, чтобы сценарии могли выполняться автоматически без ручного запуска
приложения после перезагрузки компьютера.

Если автозапуск больше не нужен, его можно отключить в настройках.


10. Папки приложения

В папке AutoScope могут быть следующие рабочие каталоги:

Databases
Здесь находятся базы данных.

Reports
Здесь находятся сформированные отчеты.

Logs
Здесь находятся журналы работы приложения и модулей.

Jobs
Здесь находятся сохраненные сценарии.

Configs
Здесь находятся настройки и конфигурационные файлы.

Parsers
Здесь находятся модули сбора данных.

Analyzers
Здесь находятся модули анализа данных.

Python
Встроенная среда Python, нужная для работы Python-модулей.

Не удаляйте эти папки без необходимости. Если вы хотите сбросить тестовые данные, обычно достаточно
очистить Databases, Reports, Logs и Jobs, но не Configs, Parsers, Analyzers и Python.


11. Что отправить разработчику при ошибке

Если приложение работает неправильно, отправьте:

1. Что именно вы делали перед ошибкой.
2. Скриншот окна или сообщения об ошибке.
3. Файлы из папки Logs, созданные во время ошибки.
4. Название выбранного парсера, анализатора или сценария.
5. Примерный текст параметров запуска, если он важен.

Это поможет быстрее понять причину проблемы.


12. Частые проблемы

Приложение не запускается.
Убедитесь, что архив полностью распакован. Не запускайте AutoScope прямо из архива.

Парсер не работает.
Проверьте, установлен ли Google Chrome. Также возможно, что сайт-источник временно ограничил доступ
или изменил структуру страниц.

Отчет не открылся автоматически.
Откройте папку Reports и запустите последний HTML-файл вручную.

Сценарии не запускаются автоматически.
Проверьте, включена ли автопроверка сценариев в настройках, задан ли интервал больше 0 и не было ли
приложение полностью закрыто через трей.

В базе нет новых данных.
Возможно, парсер не нашел подходящих объявлений, источник временно недоступен или выбранная база не та,
в которую выполнялся сбор.


13. Ограничения beta-версии

Эта версия предназначена для тестирования. Возможны:
- ошибки интерфейса;
- неполная обработка некоторых ситуаций;
- зависимость парсеров от изменений сайтов;
- временные проблемы с отдельными анализаторами;
- необходимость вручную отправлять логи разработчику.

Основная цель beta-версии — проверить приложение на разных компьютерах и понять, какие проблемы
возникают при реальном использовании.


14. Краткий сценарий проверки

1. Распакуйте AutoScope в отдельную папку.
2. Запустите AutoScope.WpfClient.exe.
3. Создайте или выберите базу данных.
4. Запустите сбор данных.
5. Дождитесь завершения процесса.
6. Запустите анализ.
7. Откройте отчет.
8. При желании создайте сценарий и проверьте его ручной запуск.
9. Если возникла ошибка, отправьте описание и логи.
